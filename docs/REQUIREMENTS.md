# AI人事24h 要件定義書 v7.1（最終確定版）

## 1. サービス概要

- **サービス名:** AI人事24h
- **種別:** B2B SaaS
- **概要:** AIが24時間365日、企業の一次面接を自動実施する。応募者はブラウザからリアルタイム音声AI面接を受け、企業は管理画面で録画・AI採点・要約を閲覧し、二次面接候補を選別する。
- **提供価値:** 面接工数削減、応募率向上（24時間対応）、統一基準による公平評価、録画・ログ保存による振り返り、テキストデータ永年蓄積によるデータ資産化、応募者への即時フィードバックによる面接体験向上。
- **関係者:** 応募者（Candidate）、企業ユーザー（Client）、運営ユーザー（Admin）の3者。

## 2. ビジネスモデル・課金

### 2-1. 初期費用
- 20万円（税別）：導入セットアップ＋質問バンク設計含む
- 職種追加：+10万円/件（職種別に質問を変えたい場合）

### 2-2. 月額プラン

| プラン | 面接件数/月 | 月額（税別） | 備考 |
|--------|-------------|-------------|------|
| A | 1〜10件 | 6万円 | 10件目まで |
| B | 11〜20件 | 12万円 | 推奨 |
| C | 21〜30件 | 18万円 | - |
| 要相談 | 31件以上 | 個別見積 | - |

### 2-3. 自動繰上げプラン
- デフォルトOFF
- ONにすると月内の面接件数が現プラン上限到達時に上位プランへ自動移行（最大30件）
- ONからOFFへの切替は翌月適用

### 2-4. プラン変更ルール
- アップグレード：即時適用
- ダウングレード：翌月適用

### 2-5. 課金カウント基準
- 面接開始後**10分超**で1件カウント
- 10分未満は0件
- ネットワーク切断・サーバー接続エラーによる中断は0件
- 不適切行為による強制終了は面接時間に関わらず0件
- 面接開始＝AIが最初の音声を出力した時点のサーバータイムスタンプ
- 面接終了＝面接終了イベントのサーバータイムスタンプ

### 2-6. 課金サイクル
- Stripe経由の月次サブスクリプション（契約日基準）
- 日割りなし、初月満額、無料トライアルなし

### 2-7. プラン上限到達時
- 自動繰上げOFFで上限到達 → 新規応募者に「現在、面接の受付を一時停止しております。恐れ入りますが、しばらく経ってから再度お試しください。」と表示
- 企業には月間上限到達通知メール送信済み

## 3. ユーザー種別と権限

### 3-1. 応募者（Candidate）
- **認証:** SMS OTP認証（Twilio Verify）
- メールアドレスはフォーム入力のみ（認証には未使用）
- **再受験制限:** 同一電話番号×同一企業で1年間不可
- **重複検知:** 氏名＋生年月日の一致でフラグ通知（ブロックはしない）
- **冷やかしロック:** 面接開始後10分未満で離脱を3回で永久ロック（ネットワーク切断・サーバーエラー・面接開始前キャンセルは対象外）
- ロック解除は運営管理画面から手動

### 3-2. 企業ユーザー（Client）
- **認証:** メールアドレス＋パスワード（Supabase Auth）
- autocomplete="email" / autocomplete="current-password" 設定
- **セッション:** 有効期限24時間
- **ブルートフォース対策:** 5回連続失敗で15分ロック、10回連続失敗で永久ロック
- **データ分離:** Supabase RLSで自社データのみ閲覧可能
- 初回ログイン時にオンボーディングツアー表示（スキップ可能）

### 3-3. 運営ユーザー（Admin）
- **認証:** メールアドレス＋パスワード＋2FA（TOTP）、2FA必須
- 全企業のデータを横断閲覧可能
- 質問バンク設計・企業設定・テキストデータ永年保管・請求管理・停止管理・セキュリティアラート操作

## 4. 応募者側機能（F-C：Candidate Functions）

### F-C-001：画面A（開始・同意画面）
- 企業ごとのランダム文字列URLからアクセス
- Google reCAPTCHA v3設置
- 「この面接は最大約40分間です。静かな環境でお受けください。」案内文表示
- 2つの同意チェックボックス（両方チェックで次へ進行可能）
  - チェック1：応募先企業への映像・音声・回答内容提供同意（録画90日保存、テキスト永年保存）
  - チェック2：運営会社へのテキストデータ保管・匿名化活用同意
- プラン上限到達時は受付停止メッセージ表示

### F-C-002：画面B（SMS認証画面）
- 電話番号入力＋OTP入力（6桁、有効期限5分）
- 「面接をキャンセルする」リンク（確認ダイアログ→データ破棄→終了画面）
- **SMS認証4層防御:**
  - 第1層（OTP入力ロック）：5回連続失敗で永久ロック
  - 第2層（SMS送信制限）：同一電話番号1日5回、同一IP1時間3回、同一IPから異なる番号1日10回、再送間隔60秒
  - 第3層（reCAPTCHA）：SMS送信ボタンにreCAPTCHA v3
  - 第4層（異常検知）：大量リクエスト検知時に運営メール通知、IPブロックは運営手動
- 再受験チェック（同一電話番号×同一企業、1年以内）
- サポートメールアドレス表示（ドメイン取得後確定）

### F-C-003：画面C（履歴書相当フォーム）
- 「面接をキャンセルする」リンク
- **入力項目:**
  - 氏名（姓・名、必須）
  - フリガナ（セイ・メイ、必須）
  - 生年月日（日付ピッカー、年齢自動算出、必須）
  - 性別（プルダウン：男性/女性/その他/回答しない、必須）
  - 電話番号（SMS認証済み自動入力・編集不可、必須）
  - メールアドレス（必須）
  - 居住都道府県（47都道府県プルダウン、必須）
  - 最終学歴（プルダウン：中学卒/高校卒/専門学校卒/短大卒/大学卒/大学院卒/その他、必須）
  - 就業形態（新卒/中途、必須）
  - 業界経験（経験あり/未経験、必須）
  - 希望職種（企業ごとプルダウン、必須）
  - 職歴・業種（テキストエリア、上限500文字、任意）
  - 保有資格（テキストエリア、上限300文字、任意）

### F-C-004：画面D（デバイス準備・通信チェック画面）
- 「面接をキャンセルする」リンク
- マイク・カメラ使用許可取得
- **通信チェック合格基準:** WebRTC ICE取得成功、音声遅延1000ms以下、カメラ映像取得成功
- **マイクテスト:** 音声入力レベルメーター表示、閾値以上の音量検出で合格
- 全チェック合格で「面接を開始する」ボタン有効化

### F-C-005：画面E（リアルタイムAI面接画面）
- **AI:** OpenAI Realtime API（gpt-4o-mini-realtime-preview）、ブラウザWebRTC
- **最大40分**、沈黙2秒でVAD発話終了判定
- **画面構成:** 中央にAIアバター（静止画＋CSS/Canvasまばたき・微揺れ・脈動）、左上に応募者カメラ映像（小窓）
- **深掘り質問:** 回答が曖昧・矛盾・10秒未満の場合、最大1回/質問
- 35分経過時にAI口頭案内「残り時間が少なくなってきましたので、最後の質問に移ります」
- **回線品質監視:** WebRTC getStats()定期取得、閾値超過時に非侵入型バナー表示（面接は継続）
- **録画:** クライアント側MediaRecorder、WebM VP9、1280×720、30fps、音声ミックス
- **録画保存:** ハイブリッド方式（10秒間隔Blob蓄積＋3分ごとIndexedDBバックアップ）
- **ネットワーク切断:** 60秒以内で自動再接続（切断中もMediaRecorder継続）、60秒超で中断扱い
- **ブラウザ閉鎖:** 中断扱い、IndexedDB残存分のみ保存、再接続不可
- **無言1分:** 自動終了
- **不適切行為:** AI検知で即時終了、「不適切行為により終了」フラグ、課金なし
- **部分レポート:** 10分超で中断時、回答済み質問のみで部分評価レポート自動生成

### F-C-005-2：画面E2（送信中・フィードバック画面）
- プログレスバー＋「画面を閉じないでください」メッセージ
- Cloudflare R2へマルチパートアップロード（5MiB以上/パート、最大3回リトライ）
- beforeunloadで閉じ防止警告
- **フィードバック3ブロック（フェードインで順次表示）:**
  - ブロック1（即時）：面接サマリー（面接時間・質問数）、クライアント側情報のみ
  - ブロック2（10〜20秒後）：強みフィードバック（2〜3個、ポジティブのみ）、OpenAI GPT API
  - ブロック3（20〜40秒後）：パーソナリティ分析（行動力/協調性/分析力/創造性/安定性の5軸5段階レーダーチャート＋コメント）
- **3つの非同期処理が同時並行:** 録画アップロード・応募者フィードバック生成・企業向けレポート生成
- アップロード完了後 → IndexedDB削除 → 画面F遷移
- 部分欠損/録画なし時は運営メール通知

### F-C-006：画面F（終了画面）
- 「面接お疲れ様でした。結果は企業の担当者よりご連絡いたします。」
- 画面E2のフィードバック3ブロックをそのまま表示
- 満足度評価（1〜5段階の星評価、任意）、運営側で保管

## 5. 面接評価・レポート生成機能（F-R：Report Functions）

### F-R-001：評価軸設計（全企業統一6軸）
1. **コミュニケーション力** — 質問の意図を正確に理解し、的確かつ分かりやすく回答できているか
2. **論理的思考力** — 回答に筋道が通っており、根拠や具体例を用いて説明できているか
3. **主体性・行動力** — 自ら考え、行動した経験や姿勢が回答から読み取れるか
4. **志望度・意欲** — 応募企業・職種への関心や熱意が回答に表れているか
5. **ストレス耐性・柔軟性** — 困難な状況や予想外の質問に対して冷静に対応できているか
6. **誠実性・一貫性** — 回答に矛盾がなく、誇張や虚偽の兆候が見られないか

### F-R-002：採点方法
- **2段階方式:** 各軸の加重平均（重み1.0〜2.0）→ 軸スコア20点満点 → 6軸合計120点 → 100点換算
- 各質問に主軸1〜2つ＋副軸、運営が紐づけ設定（企業側変更不可）

| ランク | 軸スコア | 総合スコア |
|--------|----------|-----------|
| A | 17〜20点/軸 | 85〜100点 |
| B | 13〜16点/軸 | 70〜84点 |
| C | 9〜12点/軸 | 50〜69点 |
| D | 5〜8点/軸 | 30〜49点 |
| E | 1〜4点/軸 | 1〜29点 |

### F-R-003：質問ごとの採点出力
- 評価対象の軸名、点数（20点満点）、評価ランク（A〜E）、根拠発言（原文引用）、評価理由（2〜3文）、改善ポイント（D・E評価時）

### F-R-004：レポート生成
- 面接終了後に非同期開始、録画アップロード完了を待たない
- **5分以内**の完了目標
- 完了時に企業へメール通知（管理画面リンク付き）
- 失敗時3回自動リトライ → 3回失敗で運営メール通知
- **レポート内容:** 質問ごとの採点一覧、Q&Aまとめ、要点（3〜7行）、総合評価レポート（強み・懸念点・深掘り推奨・総合所見）、全文ログ（AI/応募者区分）、録画動画

### F-R-005：部分レポート生成
- 10分超で中断時、回答済み質問のみで生成
- 「面接中断のため部分評価」フラグ付与
- 未回答質問は「未実施」表示

### F-R-006：録画保存
- Cloudflare R2 Privateバケット、**90日間保存**（ライフサイクルルール自動削除）
- 再生時は署名付きURL（有効期限10分、IP制限）、再生のたびに新規発行
- 認証済みユーザーのみ閲覧可能

### F-R-007：応募者向けフィードバック生成
- OpenAI GPT APIで1回のコールで一括生成（強み2〜3個＋パーソナリティ5軸）
- **20秒以内**の完了目標
- ポジティブのみ、採用評価とは無関係
- 企業管理画面には表示しない（画面表示のみ、保存なし）
- 満足度評価データ（1〜5段階）は運営側で永年保管

## 6. 企業管理画面機能（F-B：Business Functions）

### F-B-001：ログイン
- メールアドレス＋パスワード（Supabase Auth）
- formタグ＋submit、autocomplete設定でブラウザパスワード保存対応
- セッション24時間、5回失敗→15分ロック、10回失敗→永久ロック
- パスワードリセット：Supabase Authメールリンク方式

### F-B-001-2：オンボーディングツアー（初回のみ）
- ステップ1：応募者一覧の説明
- ステップ2：応募者詳細への遷移方法
- ステップ3：テンプレートメール送信の説明
- ステップ4：プラン管理の説明
- スキップ可能、完了/スキップ後は再表示しない

### F-B-002：応募者一覧
- 面接日が新しい順、1ページ20件、ページネーション
- **表示カラム:** 面接日、氏名、希望職種、総合評価（A〜E）、要点（3行省略＋展開）、ステータス
- **ステータス:** 未対応（黄色）/二次面接へ（緑）/不採用（グレー）
- 部分レポートは「部分評価」バッジ表示
- **フィルタ:** 日付範囲、職種、評価ランク、ステータス
- **キーワード検索:** 氏名・メールアドレス・電話番号・社内メモ
- **CSVエクスポート:** 氏名、メール、電話番号、職種、評価、ステータス、面接日（絞り込み条件適用）
- メール・電話番号カラムは一覧には非表示（個人情報保護）
- **コピーボタン:** メール・電話番号のワンクリックコピー（詳細画面で提供）

### F-B-003：応募者詳細（5タブ構成）
- **基本情報タブ:** F-C-003のフォーム全項目表示、重複応募者フラグ自動表示
- **面接録画タブ:** 720p WebM動画プレイヤー（署名付きURL、10分有効、IP制限）、90日以内何度でも再生可能、部分欠損/録画なし表示、「録画動画は面接実施日から90日後に自動削除されます」注意書き
- **AI評価タブ:** 面接サマリー（ランクA〜E＋100点換算）、6軸スコア、質問ごとの採点一覧（軸名・点数・ランク・根拠引用・評価理由・改善ポイント）、Q&Aまとめ、全文ログ（AI/応募者区分）、部分レポート時は「部分評価」フラグ表示
- **メモタブ:** 社内メモCRUD（2000文字上限、履歴保存なし）
- **ステータス変更タブ:** 「二次面接へ進む」「不採用」ボタン
- **不適切行為による終了:** 赤バナーで「不適切な行為が検知されたため、途中で終了されました」表示
- **連絡導線:** メールアドレスコピー、電話番号コピー、スマートフォンでtel:リンク発信、テンプレートメール送信ボタン
- **送信履歴:** テンプレートメールの送信日時・種別・宛先一覧

### F-B-004：テンプレートメール送信
- デフォルト2種類、企業が文面CRUD可能
- **自動挿入変数:** {応募者名}、{企業名}、{面接日}、{希望職種}
- Resendで送信
- **確認フロー:** 送信ボタン → 変数展開プレビュー → 「この内容で送信」/「戻って編集」
- **テンプレート1（二次面接のご案内）:** 件名「【{企業名}】二次面接のご案内」
- **テンプレート2（選考結果のご連絡）:** 件名「【{企業名}】選考結果のご連絡」

### F-B-005：プラン・契約管理
- 現在のプラン名、月間利用件数/上限、自動繰上げON/OFF表示
- プラン変更UI（アップグレード即時、ダウングレード翌月）
- 自動繰上げ切替UI（ONからOFFは翌月適用）

### F-B-006：請求履歴
- 月次請求一覧（対象月、プラン、カウント件数、請求金額）
- Stripe連携の請求書PDFリンク

### F-B-007：一時停止申請
- 申請日から1ヶ月後に自動停止
- 停止前であれば取り消し可能
- 申請時に運営メール通知

### F-B-008：緊急停止申請
- 運営承認後に即時停止
- 申請時に運営メール通知

### サイドバー
- 下部に面接URLコピーボタン常時表示

## 7. 運営管理画面機能（F-A：Admin Functions）

### F-A-001：ログイン
- メールアドレス＋パスワード＋2FA（TOTP）、2FA必須

### F-A-001-2：ダッシュボード
- 本日の全企業合計面接件数
- 当月の全企業合計面接件数
- 未処理セキュリティアラート数（A-10リンク付き）
- レポート生成失敗・録画アップロード失敗の未対応件数
- 今月の概算売上合計
- アクティブ企業数

### F-A-002：契約企業一覧
- 企業名、プラン、ステータス（稼働中/停止中）、当月利用件数
- 企業名クリックで詳細遷移

### F-A-003：企業詳細・切替
- 企業詳細（名前、メール、プラン、契約日、ステータス、職種、質問バンク）
- 企業間切替機能

### F-A-004：企業設定管理
- 面接URL発行・無効化・再発行（ランダム文字列）
- 希望職種プルダウン選択肢を企業ごとに設定（職種CRUD）
- 企業ON/OFF管理

### F-A-005：質問バンク管理
- 企業ごとに質問バンクCRUD
- 各質問に主軸（1〜2つ）、副軸、重み（1.0〜2.0）設定
- 質問並び順、深掘り条件有無設定
- 1企業1パターン基本、職種別は追加費用後に追加

### F-A-006：応募者テキストデータ管理
- 全企業の応募者テキストデータ（フォーム情報、面接ログ、評価レポート）を永年保管
- 検索・絞り込み・エクスポート（読取専用）

### F-A-007：請求管理
- 月次自動集計確認
- Stripe請求書自動発行管理
- 企業別請求履歴閲覧

### F-A-008：停止・再開管理
- 一時停止申請確認
- 緊急停止申請の承認・却下
- 強制ON/OFF切替（再活性化）

### F-A-009：企業結果の運営反映

## 9. セキュリティ設計（F-S：Security Functions）

### F-S-001：通信暗号化
- HTTPS（TLS 1.2以上）、HSTS、CSP
- WebRTC音声・映像はDTLS-SRTPで暗号化

### F-S-002：認証・認可
- 企業：Supabase Auth メール＋パスワード
- 運営：Supabase Auth メール＋パスワード＋TOTP
- 応募者：Twilio Verify SMS OTP
- 全テーブルにSupabase RLS適用

### F-S-003：パスワードセキュリティ
- bcryptハッシュ（Supabase Auth標準）
- パスワードリセット：メールリンク方式

### F-S-004：SMS認証セキュリティ（4層防御）
1. reCAPTCHA v3
2. IP制限（1時間3回/IP、1日10回/IP異番号）
3. Twilio Rate Limit（同一番号1日5回、再送60秒間隔）
4. 冷やかしロック（3回で永久）

### F-S-005：ブルートフォース対策
- 企業：5回→15分ロック、10回→永久ロック
- 運営：同上＋2FA必須
- 解除は運営管理画面A-10から手動

### F-S-006：動画アクセス制御
- R2 Privateバケット、署名付きURL（10分有効、IP制限）
- 認証済みユーザーのみ、アクセスログ記録

### F-S-007：冷やかし対策
- 10分未満離脱を3回で永久ロック（ネットワーク切断・サーバーエラー・開始前キャンセル対象外）

### F-S-008：面接URL不正拡散対策
- ランダム文字列URL、reCAPTCHA v3、運営からURL無効化・再発行可能

### F-S-009：不適切行為対策
- 無言1分自動終了、暴言検知即時終了、「不適切行為により終了」フラグ、課金なし

### F-S-010：再受験・重複対策
- 同一電話番号×同一企業で1年間再受験不可
- 氏名＋生年月日の重複検知→フラグ通知（ブロックなし）
